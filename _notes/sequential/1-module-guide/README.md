What is a module:
- a module is a directory
- that has routes file
- and controllers
- as well as database migrations and seeders

Usage:
- a module can be copied to any project (using simple guide in its README.md)
- developer can run migrations and seeders
- and add routes and controllers to their project
- and it is ready to work in there.
- `modules/translator/MODULEINFO.md` has module info. compact short human redable (with endpoints, input for endpoints and output)
- also `moduleinfo.json` helps the /app for methods, endpoints etc.

Stack, languages and style;
- for understanding stack, language and directory structure follow info at skill `.agents/skills/api-dev`
- regarding programming style, use `programming-style` and `software architecture` from `.agents/skills` as well
